Semantic Model

The semantic model is split into three modules.

  • Reactive Programs
  • Conditional Safety
  • Explicit Product

Reactive Programs

SpecV2.ReactiveModel defines the extensional notion of reactive program used in the development. One synchronous tick is total by construction through the fields select and select_enabled.

Conditional Safety

SpecV2.ConditionalSafety adds the specification layer:

  • an assumption automaton over inputs;
  • a guarantee automaton over input/output observations;
  • the semantic predicates AvoidA and AvoidG.

Explicit Product

SpecV2.ExplicitProduct synchronizes the program and both automata. This is the level where global violations become local dangerous steps.
