Fine-grained backend families emitted by the implementation.

They are later projected onto:

  • semantic clause families (`no_bad`, `initial_goal`, `user_invariant`, `automaton_support`);
  • proof layers (`safety`, `helper`);
  • helper phases (`init_goal`, `propagation`).

This mirrors the current Rocq/paper architecture where the four semantic labels classify generated clauses, while the actual proof split is `Safety` versus `Helper`.
